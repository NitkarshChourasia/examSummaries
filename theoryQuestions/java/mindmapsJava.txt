[2023-02-21 09:50:17] RESPONSE: 
Delegation Event Model in Java

- Event Source: Generates events and passes them to the listener
- Event Listener: The interface that receives the event
- Event Object: Contains information about the event
- Event Handler: Code to respond to the event
- Delegation: The process of passing the event to the listener object 
- Advantages:
    - Simplicity: Easy to understand and use
    - Flexibility: Events can be handled in different ways
    - Separation of Concerns: Event source and listener are separate 
- Disadvantages:
    - Code Complexity: Event handler code can become complex 
    - Performance: Delegation may be slower than traditional event handling

[2023-02-21 09:56:45]: Create a mindmap for the following question: In the context of Java and Abstract Window Toolkit, What is the use of adapter class in Java? Explain any one of the adapter classes defined in Java.

[2023-02-21 09:57:00] RESPONSE: 
Adapter Class in Java & Abstract Window Toolkit

• Use: 
  - To provide a different interface for existing classes
  - To provide a bridge between unrelated interfaces
  - To allow classes to work together that normally couldn’t

• Adapter Classes Defined in Java
  - WindowAdapter: A class that implements WindowListener interface, with empty methods for all the window events.
  - KeyAdapter: A class that implements KeyListener interface, with empty methods for all the key events.
  - MouseAdapter: A class that implements MouseListener interface, with empty methods for all the mouse events.
  - FocusAdapter: A class that implements FocusListener interface, with empty methods for all the focus events.
  - ComponentAdapter: A class that implements ComponentListener interface, with empty methods for all the component events.
  - ContainerAdapter: A class that implements ContainerListener interface, with empty methods for all the container events.
  - ActionAdapter: A class that implements ActionListener interface, with empty methods for all the action events.

• Example:
  - WindowAdapter
    - Used to handle window events like window opening, closing, iconification and deiconification, window activation and deactivation, etc. 
    - It has empty implementations for all the window events. 
    - We need to override the methods for which we need to handle the events.
